Đề tài: Phần mềm Quản lý Thư viện

1. Khảo sát hệ thống
  • Mục tiêu: Hiểu quy trình mượn/trả hiện tại và khó khăn gặp phải (mất sách, tìm lâu).
  • Hoạt động: Phỏng vấn thủ thư, quan sát thực tế.
  • Công cụ: Phiếu khảo sát.
2. Phân tích yêu cầu
  • Mục tiêu: Xác định chức năng cần có (Mượn, Trả, Quản lý sách, Độc giả).
  • Hoạt động: Vẽ sơ đồ Use Case, chốt yêu cầu với khách hàng.
  • Công cụ: Sơ đồ Use Case, Tài liệu SRS.
3. Thiết kế
  • Mục tiêu: Xây dựng kiến trúc hệ thống, CSDL và giao diện.
  • Hoạt động: Thiết kế bảng dữ liệu (Sách, Phiếu mượn) và vẽ màn hình giao diện.
  • Công cụ: Draw.io (vẽ sơ đồ), Figma (giao diện).
4. Code (Thực hiện)
  • Mục tiêu: Viết mã nguồn để phần mềm chạy được.
  • Hoạt động: Viết code xử lý logic (tính tiền phạt) và kết nối CSDL.
  • Công cụ: VS Code, SQL Server.
5. Kiểm thử
  • Mục tiêu: Tìm lỗi (bug) để đảm bảo phần mềm đúng yêu cầu.
  • Hoạt động: Chạy thử các trường hợp (nhập sai, mượn quá hạn) để tìm lỗi.
  • Công cụ: Excel (viết Test case).
Câu 2: Quy trình (Scrum, Waterfall, Thác Nước...) có mấy giai đoạn, vai trò và ý nghĩa của từng giai đoạn. Công cụ nào hỗ trợ làm nhóm, mô tả, ý nghĩa.

A. Quy trình Scrum

Scrum là quy trình linh hoạt (Agile), chia dự án thành các chu kỳ ngắn gọi là Sprint.

1. Các vai trò (Roles):

  • Product Owner: Quản lý yêu cầu và giá trị sản phẩm.
  • Scrum Master: Quản lý quy trình, loại bỏ trở ngại.
  • Dev Team: Nhóm phát triển (Code, Test, Design).

2. Các giai đoạn (Sự kiện) trong Sprint:

  • Sprint Planning: Lập kế hoạch, chọn việc làm trong Sprint.
  • Daily Scrum: Họp nhanh 15p mỗi ngày để báo cáo tiến độ.
  • Sprint Review: Demo sản phẩm cuối kỳ cho khách hàng xem.
  • Sprint Retrospective: Nhóm họp rút kinh nghiệm để cải tiến.

B. Công cụ hỗ trợ nhóm

1. Trello / Jira (Quản lý công việc)

  • Mô tả: Bảng quản lý dạng thẻ (Kanban).
  • Ý nghĩa: Giúp biết ai làm gì, tiến độ đến đâu, tránh sót việc.

2. Git / GitHub (Quản lý mã nguồn)

  • Mô tả: Hệ thống lưu trữ và quản lý phiên bản code.
  • Ý nghĩa: Giúp nhiều người code chung không bị đè file, lưu lại lịch sử sửa đổi.
